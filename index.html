Terminal Commands (v39): Functions

Task 1: Write a function that takes 0 or more numerical values and prints out the sum of them. The function should print "0" if no paramerters were passed. You may assume that all parameters are numerical, this doesn't need to be checked (but you may do so if you wish).

Now, to the explaining part.

Just like any other language, bash has its own strange syntax, and when it comes to functions you may define one in one of two ways:

function test { }

test () { }

Other than that you may use a whole range of expressions. I'm using -eq which is basically the same as ==. $# stands for the number of params that were passed.

One strange but kind of nice thing is that bash by default in for's loops over params that were passed. Always nice with short code.

sum=$(($sum+$i)), here we are setting sum, our total value, equal to an expression where we are adding the value of sum plus our current number we are looping over.

What the regexes mean:

LiU: We are searching for 4 to 5 characters of type A-Z and a-z. Then we expect 2 to 3 digits.

Date: We're searching for 4 digits followed by dash, 2 digits, dash, followed by 2 more digits.

Date (with check): We're searching for a year (4 digits), it could be now, it could be in the future. Dash. If the following digit is 1, only allow 0-2 after it. If the following digit is 0, allow 0-9 after it. Dash. The next digit must either be in range 0-2 or be a 3, if in range 0-2, we'll allow 0-9 after it, if it's a 3, only allow 0 or 1. In Python "pattern.findall()" will return a tuple if the regex has multiple capture groups, so the whole thing is surrounded with "()" to return our whole match as well.

HTML: We're searching for a left-angled bracket that has a series of words after it. Python will only return the capture group with "pattern.findall()".

Regex was created for pattern matching in regular languages, and HTML is a context-free language. Therefore it is extremely hard to write a regex to catch all types and variations of HTML. In need of an HTML parser, it is recommended to use a library especially designed for HTML parsing as it would be much faster and easier to use than to maintain a homemade HTML regex.
